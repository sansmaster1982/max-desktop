# Вход и безопасность аккаунта

Как входить в MAX Desktop и как не словить временную блокировку номера. Это
неофициальный клиент — у строгого мессенджера сторонний доступ всегда несёт
риск флага антифродом. Ниже — что снижает риск на практике.

## Как войти

### Вариант 1 — по веб-токену (рекомендуется)

Меньше всего сигналов для антифрода: токен берётся из вашей же веб-сессии, без
повторных SMS и без выдачи себя за Android.

1. Откройте `web.max.ru`, войдите в аккаунт.
2. `F12` → вкладка **Console**, выполните:

   ```js
   Object.entries(localStorage).filter(([k,v])=>v.length>40).forEach(([k,v])=>console.log(k,'=>',v))
   ```

   Среди выведенных значений будет auth-token (длинная строка ~700 символов из
   букв/цифр/`_-`). Если в `localStorage` пусто — посмотрите `F12` → **Application**
   → Local Storage / Session Storage / IndexedDB для `web.max.ru`.
3. Скопируйте значение токена целиком (без пробелов и переносов).
4. В приложении: вкладка **По токену** → вставьте → **Войти по токену**.

Режим сам ставит `deviceType=WEB` — иначе сервер веб-токен не принимает. Токен
живёт недолго; перестал пускать — возьмите свежий тем же способом.

### Вариант 2 — по SMS

1. Вкладка **По SMS** → номер в формате `+79991234567` → **Получить код**.
2. Введите код из SMS. Если включён 2FA — введите пароль 2FA.

Важно: повторный запрос SMS доступен не чаще раза в 45 c (защита от блокировки).
Не запрашивайте код многократно и не вводите 2FA наугад — после нескольких
неверных попыток сервер временно блокирует.

После любого входа токен сохраняется локально в
`%APPDATA%\MAX Desktop\max_auth_token.txt`, дальше вход автоматический.

## Как НЕ словить блокировку

Блокировка — это срабатывание защиты аккаунта MAX на признаки «угон/бот». Чтобы
не триггерить:

- **Стабильное устройство.** Не удаляйте `%APPDATA%\MAX Desktop\device_id.txt` —
  это «отпечаток устройства». Если его стереть, для сервера это новое устройство.
  (Клиент сам бережёт этот файл, в том числе при выходе.)
- **Не гоняйте токен с разных устройств.** Один токен, появляющийся с нескольких
  разных `deviceId` за короткое время, — главный сигнал угона. НЕ запускайте
  диагностические скрипты/второй клиент с тем же токеном.
- **Один экземпляр.** Не держите несколько копий приложения залогиненными
  одновременно.
- **Не частите авторизацией.** Не дёргайте SMS повторно, не перебирайте 2FA.
- **Меньше входов по SMS — больше по токену.** SMS-флоу выдаёт себя за Android и
  шлёт SMS; веб-токен спокойнее.
- **Постоянный IP/гео.** По возможности без VPN при входе и в сессии — скачки
  гео повышают риск-скоринг.
- **Свежая версия.** В Настройки → версия держите `app_version` близкой к текущей
  версии официального приложения.

## Если номер уже заблокировали

1. Войдите один раз **официальным приложением** MAX с этого номера —
   восстановить «доверие» к устройству/номеру.
2. Переждите временный блок (обычно снимается сам через время).
3. Дальше пользуйтесь этим клиентом аккуратно: по веб-токену, со стабильным
   `device_id.txt`, без повторных SMS и без сторонних скриптов на аккаунте.

Разблокировать со стороны клиента нельзя — это решает только сервер MAX.

## Что в клиенте уже сделано для снижения риска

- **Keepalive (главное).** PING каждые 20 c держит соединение живым. Без него
  сервер рвал простаивающий сокет за 11–61 c, клиент переподключался с новым
  входом каждые ~30 c — это «шторм сессий», который и ловил антифрод. Проверено:
  с keepalive соединение живёт >70 c без переподключений.
- Гашение реконнекта: база 5 c + джиттер вместо 2 c (если реальный разрыв всё же
  случился — не частим повторными входами).
- Стабильный `deviceId` в отдельном файле: создаётся один раз, переживает выход и
  сброс настроек.
- Одно соединение: новый вход закрывает старый сокет (без параллельных сессий).
- Кулдаун 45 c на повторный запрос SMS.

## Честная оговорка

Эти меры убирают грубые сигналы, но не делают сторонний клиент неотличимым от
официального. Подделывать сетевой отпечаток под оф. приложение — это обход
средств защиты, такого тут нет. Поэтому остаточный риск флага сохраняется;
минимизируется он именно поведением выше.
